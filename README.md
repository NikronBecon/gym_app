# ЖимЖим

Личное iPhone-приложение для планирования тренировок, ведения подходов и отслеживания прогресса.

> GitHub хранит исходный код — его нельзя просто скачать и установить на обычный iPhone как приложение. Для личного использования есть два пути: запустить приложение через Xcode на своём iPhone или выложить сборку в TestFlight.

## Что умеет приложение

- создавать и редактировать шаблоны тренировок;
- вести тренировку: подходы, вес, повторения, таймер отдыха и заметки;
- добавлять упражнение прямо во время тренировки;
- работать с каталогом из 100 упражнений на русском языке и GIF-анимациями;
- планировать тренировки в календаре и получать локальные напоминания;
- хранить историю завершённых тренировок;
- отслеживать вес тела, объём, рабочие веса, время в зале и серию тренировок.

## Требования

- Mac с актуальной версией Xcode;
- iPhone или симулятор с iOS 17 и новее;
- Apple Account для установки на личный iPhone;
- членство в Apple Developer Program — только если нужна установка через TestFlight или распространение приложения.

## Запуск на Mac или в симуляторе

```bash
git clone https://github.com/NikronBecon/gym_app.git
cd gym_app
open GymProgress.xcodeproj
```

В Xcode:

1. Вверху выбери схему `GymProgress` и любой iPhone Simulator.
2. Нажми кнопку запуска ▶ или `⌘R`.
3. В симуляторе приложение будет отображаться как **ЖимЖим**.

## Установка на свой iPhone через Xcode

Это бесплатный и самый быстрый способ поставить приложение только себе.

1. На iPhone включи **Режим разработчика**: `Настройки → Конфиденциальность и безопасность → Режим разработчика`. Телефон перезагрузится; после перезагрузки подтверди включение и введи код-пароль.
2. Подключи iPhone к Mac кабелем и разблокируй его. Если появится вопрос, подтверди доверие к компьютеру.
3. Открой `GymProgress.xcodeproj` в Xcode.
4. Вверху, рядом с кнопкой ▶, выбери свой iPhone как устройство запуска.
5. В левой панели выбери проект **GymProgress** → target **GymProgress** → **Signing & Capabilities**.
6. В поле **Team** выбери свой Apple Account (`Personal Team`). Если аккаунта нет в списке: **Xcode → Settings → Accounts → +**.
7. Если Xcode попросит изменить Bundle Identifier, согласись и укажи уникальное значение, например `com.<твой-логин>.ZhimZhim`.
8. Нажми ▶ или `⌘R`. После первой сборки иконка **ЖимЖим** появится на iPhone.
9. Если iPhone не открывает приложение, перейди в `Настройки → Основные → VPN и управление устройством`, выбери свой Apple ID в разделе **Developer App** и нажми «Доверять».

С бесплатной Personal Team приложение работает для личного тестирования, но профиль установки действует 7 дней: затем достаточно снова подключить телефон и запустить проект из Xcode. Ничего удалять и заново настраивать не нужно; сохранённые тренировки останутся. Это официальное ограничение Apple для бесплатной учётной записи. [Подробнее у Apple](https://developer.apple.com/help/account/basics/about-your-developer-account/)

## Установка через TestFlight

TestFlight удобнее, если хочется устанавливать **ЖимЖим** как обычную тестовую программу без подключения к Mac. Для этого нужно платное членство Apple Developer Program (Apple указывает стоимость 99 USD в год или эквивалент в локальной валюте).

1. Зарегистрируйся в [Apple Developer Program](https://developer.apple.com/support/compare-memberships/).
2. В Xcode выбери **Product → Archive**.
3. В окне Organizer нажми **Distribute App → App Store Connect → Upload** и закончи загрузку.
4. В [App Store Connect](https://appstoreconnect.apple.com/) создай приложение с тем же Bundle Identifier.
5. Открой вкладку **TestFlight**, дождись обработки сборки и добавь себя как тестировщика.
6. Установи на iPhone приложение **TestFlight** из App Store, прими приглашение и нажми «Установить».

Одна сборка TestFlight доступна 90 дней; для продолжения тестирования нужно загрузить новую. [Условия TestFlight у Apple](https://developer.apple.com/help/app-store-connect/reference/app-build-statuses/)

## Публикация проекта в GitHub

Репозиторий уже настроен на:

```text

```

Перед первой отправкой проверь, что не добавляешь личные сертификаты, профили подписывания или ключи. `.gitignore` уже исключает служебные файлы Xcode.

```bash
cd /путь/к/gym_app
git status
git add GymProgress GymProgress.xcodeproj GymProgressTests GymProgressUITests README.md .gitignore
git commit -m "Подготовить ЖимЖим к публикации"
git push origin main
```

Если ветвь называется не `main`, посмотри её через `git branch --show-current` и подставь это имя в последнюю команду. Если GitHub запросит вход, используй GitHub Desktop, SSH-ключ или Personal Access Token — обычный пароль GitHub для Git по HTTPS не используется.

## Данные и приватность

- Все данные приложения хранятся локально в SwiftData внутри песочницы iPhone.
- Серверов, аккаунтов, рекламы и сетевых запросов нет.
- После обычной перезагрузки телефона данные останутся.
- Удаление приложения удалит локальную историю.
- В текущей версии данные исключены из резервной копии iCloud, а синхронизация между устройствами и экспорт/импорт пока не реализованы.

## Структура проекта

```text
GymProgress/
  App/             запуск приложения и зависимости
  Features/        экраны: тренировка, шаблоны, календарь, прогресс
  Models/          SwiftData-модели
  Services/        расчёты, уведомления, каталог упражнений
  Resources/       GIF, иконка и данные упражнений
GymProgressTests/  модульные тесты
GymProgressUITests/ UI-тесты
```

## Медиа и данные упражнений

Метаданные каталога основаны на [hasaneyldrm/exercises-dataset](https://github.com/hasaneyldrm/exercises-dataset). GIF принадлежат Gym Visual. Если репозиторий или приложение будет распространяться публично, перед этим нужно отдельно проверить права и условия использования медиа.
